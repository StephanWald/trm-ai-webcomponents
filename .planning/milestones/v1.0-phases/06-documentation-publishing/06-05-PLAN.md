    The following steps require human action because they involve account authentication and secret management that Claude cannot perform:

    **Step 1: Create GitHub repository**
    - Go to https://github.com/new
    - Create repository: `trm-ai-webcomponents` under the appropriate organization
    - Do NOT initialize with README (repo already has content)

    **Step 2: Add GitHub remote and push**
    ```bash
    git remote add origin https://github.com/<org>/trm-ai-webcomponents.git
    git push -u origin master
    ```

    **Step 3: Configure GitHub Pages**
    - Go to repo Settings > Pages
    - Under "Build and deployment", set Source to "GitHub Actions"
    - The `docs-deploy.yml` workflow will deploy automatically on next push to master

    **Step 4: Configure NPM_TOKEN secret**
    - Create an npm automation token at https://www.npmjs.com/settings/<user>/tokens
    - Token type: Granular Access Token with read+write for `@skillspilot/webcomponents`
    - Go to repo Settings > Secrets > Actions
    - Add secret: `NPM_TOKEN` with the token value

    **Step 5: Trigger first publish**
    - Option A: Create a changeset (`npx changeset`), commit, push, and merge the resulting version PR
    - Option B: Manually publish: `npm run build && npm publish --access public`

    **Step 6: Verify CDN availability**
    - After publish, verify: `https://cdn.jsdelivr.net/npm/@skillspilot/webcomponents/dist/skillspilot/skillspilot.esm.js`
    - And: `https://unpkg.com/@skillspilot/webcomponents/dist/skillspilot/skillspilot.esm.js`

    **Step 7: Update Docusaurus config (optional)**
